# Add GeoIP Enrichment to Vector Components
---

## We need to Signup at **maxmind.com** and download GeoLite2-City.mmdb
- Sign up at https://www.maxmind.com and generate license key.

## On Debian:
Download tarball
```
cd /tmp
wget "https://download.maxmind.com/app/geoip_download?edition_id=GeoLite2-City&license_key=YOUR_LICENSE_KEY&suffix=tar.gz" -O GeoLite2-City.tar.gz
```
Extract and copy mmdb to vector directory
```
tar -xzf GeoLite2-City.tar.gz
cp GeoLite2-City_*/GeoLite2-City.mmdb /etc/vector/GeoLite2-City.mmdb
```
Assign Permissions
```
chown root:vector /etc/vector/GeoLite2-City.mmdb
chmod 644 /etc/vector/GeoLite2-City.mmdb
```

**Optional weekly update:**  
- Install geoipupdate, configure /etc/GeoIP.conf with AccountID/LicenseKey/EditionIDs, run geoipupdate.
- Add cron: 0 3 * * 0 /usr/bin/geoipupdate.
---

- For airgaped infrastructure download and update mmdb file manually at your scheduled updates.
---

## Update vector.yaml
- Add on top (under data_dir: /var/lib/vector)
```
enrichment_tables:
  geoip_city:
    type: geoip
    path: "/etc/vector/GeoLite2-City.mmdb"
```  

**pfSense Enrichment**
- Add in transforms after pfsense_filterlog:
```
 pfsense_geoip:
    type: remap
    inputs: ["pfsense_filterlog"]
    source: |
      if exists(.source.ip) {
        geo, err = get_enrichment_table_record("geoip_city", { "ip": .source.ip })
        if err == null {
          if exists(geo.city_name)    { .source.geo.city_name        = geo.city_name }
          if exists(geo.country_name) { .source.geo.country_name     = geo.country_name }
          if exists(geo.country_code) { .source.geo.country_iso_code = geo.country_code }
          if exists(geo.region_name)  { .source.geo.region_name      = geo.region_name }
          if exists(geo.region_code)  { .source.geo.region_iso_code  = geo.region_code }
          if exists(geo.latitude) && exists(geo.longitude) {
            .source.geo.location = {
              "lat": to_float!(geo.latitude),
              "lon": to_float!(geo.longitude)
            }
          }
        }
      }
      if exists(.destination.ip) {
        geo, err = get_enrichment_table_record("geoip_city", { "ip": .destination.ip })
        if err == null {
          if exists(geo.city_name)    { .destination.geo.city_name        = geo.city_name }
          if exists(geo.country_name) { .destination.geo.country_name     = geo.country_name }
          if exists(geo.country_code) { .destination.geo.country_iso_code = geo.country_code }
          if exists(geo.region_name)  { .destination.geo.region_name      = geo.region_name }
          if exists(geo.region_code)  { .destination.geo.region_iso_code  = geo.region_code }
          if exists(geo.latitude) && exists(geo.longitude) {
            .destination.geo.location = {
              "lat": to_float!(geo.latitude),
              "lon": to_float!(geo.longitude)
            }
          }
        }
      }
```

**Update es_pfsense inputs in sinks with ["pfsense_geoip"]**
```
es_pfsense:
    type: elasticsearch
    inputs: ["pfsense_geoip"]
```

## Add Elasticsearch Index Templates
**Navigate to Dev Tools - Kibana**
Step | Action
-----|-------
1 | Click the hamburger menu (☰)
2 | Go to Management → Dev Tools

## Make 3 POST requests to create the template for each component.  
- Replace INDEX with logs-pfsense.filterlog-default, logs-pfsense.pfblockerng-default, logs-wazuh.alerts-default for each request:

```
PUT _index_template/INDEX-template
{
  "index_patterns": ["INDEX*"],
  "template": {
    "mappings": {
      "properties": {
        "source": {
          "properties": {
            "geo": {
              "properties": {
                "location": { "type": "geo_point" }
              }
            }
          }
        },
        "destination": {
          "properties": {
            "geo": {
              "properties": {
                "location": { "type": "geo_point" }
              }
            }
          }
        }
      }
    }
  }
}
```

## Final Steps
- Delete old indices. Find indices by GET request at Dev tools - GET /_cat/indices
- Restart Vector
- Monitor Vector logs for any errors: sudo journalctl -u vector -f
- Trigger events with public IPs and check results.
