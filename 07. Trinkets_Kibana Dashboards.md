## Prerequisites

Before creating the dashboards ensure these data views exist in Kibana.

**Go to:** Stack Management > Data Views > Create data view

|Name|Index pattern|Timestamp field|
|---|---|---|
|pfSense Filterlog|`logs-pfsense.filterlog*`|@timestamp|
|pfBlockerNG|`logs-pfsense.pfblockerng*`|@timestamp|
|Suricata|`logs-suricata*`|@timestamp|
|Wazuh Alerts|`logs-wazuh.alerts*`|@timestamp|
|Zeek|`zeek-*`|@timestamp|


## Dashboard 1: SOC Overview

![SOC_Overview](https://github.com/user-attachments/assets/830d0dc5-54f6-4b11-bbff-1ed606e088ad)

**Purpose:** Single-pane-of-glass for overall SIEM health and alert volumes across all sources.

**Go to:** Analytics > Dashboards > Create dashboard

### Panel 1.1 - Total Alerts by Source

1. Click **Create visualization**
2. Visualization type dropdown: select **Metric**
3. Data view: **Wazuh Alerts**:
4. Primary metric: **Count**
5. Click the metric label text "Count of records" > rename to **Wazuh Alerts**
6. Click **Save and return**

Repeat for **Suricata** data view, **pfSense** data view and **pfBlockerNG** data view. Arrange all 4 metrics in a single row at the top.


### Panel 1.2 - Alert Volume Over Time

1. Click **Create visualization**
2. Visualization type: **Line**
3. Data view: **Wazuh Alerts**
4. Horizontal axis:
    - Click **Add or drag-and-drop a field** > choose **@timestamp**
    - Interval: **Auto**
5. Vertical axis:
    - **Count of records**
6. To add Suricata as a second layer:
    - Click **Layer actions** > **Add layer**
    - Select **New visualization layer**
    - Data view: **Suricata**
    - Horizontal axis: **@timestamp**
    - Vertical axis: **Count of records**
7. Click the color for each layer to distinguish theme
8. Click **Save and return**


### Panel 1.3 - Top 10 Suricata Signatures

1. Click **Create visualization**
2. Visualization type: **Bar horizontal**
3. Data view: **Suricata**
4. Vertical axis:
    - Click field > select **rule.name**
    - Function: **Top values**
    - Number of values: **10**
5. Horizontal axis:
    - **Count of records**
6. Click **Save and return**


### Panel 1.4 - Wazuh Alerts by Mitre Severity

1. Click **Create visualization**
2. Visualization type: **Donut**
3. Data view: **Wazuh Alerts**
4. Slice by:
    - Click field > select **wazuh.rule.mitre.attack**
    - Function: **Top values**
    - Number of values: **10**
5. Metric: **Count of records**
6. Click **Save and return**


### Panel 1.5 - MITRE ATT&CK Tactic Distribution

1. Click **Create visualization**
2. Visualization type: **Treemap**
3. Data view: **Wazuh Alerts**
4. Group by:
    - First group: click field > **wazuh.rule.mitre.tactic**
        - Function: **Top values**
        - Number of values: **12**
    - Click **Add** to add a sub-group: **wazuh.rule.mitre.technique**
        - Function: **Top values**
        - Number of values: **5**
5. Metric: **Count of records**
6. Click **Save and return**


### Panel 1.6 - Top Blocked IPs

1. Click **Create visualization**
2. Visualization type: **Table**
3. Data view: **pfSense Filterlog**
4. Add filter in query bar: `event.action : "block"`
5. Columns:
    - Rows: **source.ip** > Top values > 15
    - Metric 1: **Count of records** (rename to "Blocks")
    - Add column: **source.geo.country_name** > Top values > 1 (rename to "Country")
6. Click **Save and return**

**Save the dashboard** as **SOC Overview**.

## Dashboard 2: Threat Hunting - Network & DNS

![Threat_Hunting](https://github.com/user-attachments/assets/4195a069-1b82-451f-80f9-18bf805f8572)


### Panel 2.1 - Protocol Distribution

1. Click **Create visualization**
2. Visualization type: **Pie - Donut**
3. Data view: **Zeek**
4. Slice by: **network.protocol.keyword** > Top values > 10
5. Metric: **Count of records**
6. Click **Save and return**


### Panel 2.2 -- DNS Query Volume Over Time

1. Click **Create visualization**
2. Visualization type: **Area**
3. Data view: **Zeek**
4. Add filter: `event.dataset.keyword : "zeek.dns"`
5. Horizontal axis: **@timestamp**
6. Vertical axis: **Count of records**
7. Breakdown: **qtype_name** > Top values > 5
8. Click **Save and return**


### Panel 2.3 - Top Blocked Domains

1. Click **Create visualization**
2. Visualization type: **Bar horizontal**
3. Data view: **pfBlockerNG**
4. Add filter: `pfblocker.type : "dnsbl"`
5. Vertical axis: **destination.domain** > Top values > 15
6. Horizontal axis: **Count of records**
7. Click **Save and return**


### Panel 2.4 - Top External Destinations

1. Click **Create visualization**
2. Visualization type: **Bar horizontal**
3. Data view: **Zeek**
4. Add filter: `NOT destination.ip.keyword : 10.* AND NOT destination.ip.keyword : 192.168.* AND NOT destination.ip.keyword : 172.16.*`
5. Vertical axis: **destination.ip.keyword** > Top values > 15
6. Horizontal axis: **Count of records**
7. Click **Save and return**


### Panel 2.5 - TLS Certificate Anomalies

1. Click **Create visualization**
2. Visualization type: **Table**
3. Data view: **Suricata**
4. Add filter: `tls.server.subject : *`
5. Columns:
    - Rows: **tls.client.server_name** > Top values > 15
    - Add column: **tls.server.x509.issuer.common_name** > Top values > 1
    - Metric: **Count of records**
6. Click **Save and return**


### Panel 2.6 -- Large Outbound Transfers

1. Click **Create visualization**
2. Visualization type: **Table**
3. Data view: **Zeek**
4. Add filter for conn logs with large transfers: use the query bar
5. Columns:
    - Rows: **source.ip.keyword** > Top values > 10
    - Add column: **destination.ip.keyword** > Top values > 1
    - Metric: **Sum of source.bytes**
    - Add column: **service** > Top values > 1
6. Sort by "Bytes Out" descending
7. Click **Save and return**


### Panel 2.7 - Top Queried Domains

1. Click **Create visualization**
2. Visualization type: **Table**
3. Data view: **Zeek**
4. Filter for DNS logs
5. Columns:
    - Rows: **zeek.dns.query.keyword** > Top values > 20
    - Metric: **Count of records**
    - Add column: **source.ip.keyword** > Top values > 1
6. Click **Save and return**


### Panel 2.8 - Threat Origin Map
This map shows which countries are generating the most firewall blocks.

1. Click **Create visualization**
2. Add > New panel **Maps**
3. Click **Add layer**
4. Select **Choropleth**
5. **Boundaries source:**
    - EMS boundaries: **World Countries**
    - Join field: **ISO 3166-1 alpha-2 code**
6. **Statistics source:**
    - Data view: **pfSense Filterlog**
    - Join field: **source.geo.country_iso_code**
7. Click **Add and continue**
8. **Layer style:**
    - Fill color: select a color ramp (light to dark)
9. **Tooltip fields:** click **+ Add** > select **name**
10. Click **Keep changes**
11. Click **Save and return**

**Save the dashboard** as **Threat Hunting - Network & DNS**.
