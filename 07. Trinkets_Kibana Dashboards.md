## Prerequisites

Before creating the dashboards ensure these data views exist in Kibana.

**Go to:** Stack Management > Data Views > Create data view

|Name|Index pattern|Timestamp field|
|---|---|---|
|pfSense Filterlog|`logs-pfsense.filterlog*`|@timestamp|
|pfBlockerNG|`logs-pfsense.pfblockerng*`|@timestamp|
|Suricata|`logs-suricata*`|@timestamp|
|Wazuh Alerts|`logs-wazuh.alerts*`|@timestamp|
|Zeek|`zeek-*`|@timestamp|


## Dashboard 1: SOC Overview

![SOC_Overview](https://github.com/user-attachments/assets/830d0dc5-54f6-4b11-bbff-1ed606e088ad)

**Purpose:** Single-pane-of-glass for overall SIEM health and alert volumes across all sources.

**Go to:** Analytics > Dashboards > Create dashboard

### Panel 1.1 - Total Alerts by Source (Metric)

1. Click **Create visualization**
2. Visualization type dropdown: select **Metric**
3. Data view: **Wazuh Alerts**:
4. Primary metric: **Count**
5. Click the metric label text "Count of records" > rename to **Wazuh Alerts**
6. Click **Save and return**

Repeat for **Suricata** data view, **pfSense** data view and **pfBlockerNG** data view. Arrange all 4 metrics in a single row at the top.

### Panel 1.2 - Alert Volume Over Time (Line Chart)

1. Click **Create visualization**
2. Visualization type: **Line**
3. Data view: **Wazuh Alerts**
4. Horizontal axis:
    - Click **Add or drag-and-drop a field** > choose **@timestamp**
    - Interval: **Auto**
5. Vertical axis:
    - **Count of records**
6. To add Suricata as a second layer:
    - Click **Layer actions** > **Add layer**
    - Select **New visualization layer**
    - Data view: **Suricata**
    - Horizontal axis: **@timestamp**
    - Vertical axis: **Count of records**
7. Click the color for each layer to distinguish theme
8. Click **Save and return**

### Panel 1.3 - Top 10 Suricata Signatures (Horizontal Bar)

1. Click **Create visualization**
2. Visualization type: **Bar horizontal**
3. Data view: **Suricata**
4. Vertical axis:
    - Click field > select **rule.name**
    - Function: **Top values**
    - Number of values: **10**
5. Horizontal axis:
    - **Count of records**
6. Click **Save and return**

### Panel 1.4 - Wazuh Alerts by Mitre Severity (Donut)

1. Click **Create visualization**
2. Visualization type: **Donut**
3. Data view: **Wazuh Alerts**
4. Slice by:
    - Click field > select **wazuh.rule.mitre.attack**
    - Function: **Top values**
    - Number of values: **10**
5. Metric: **Count of records**
6. Click **Save and return**


### Panel 1.5 - MITRE ATT&CK Tactic Distribution (Treemap)

1. Click **Create visualization**
2. Visualization type: **Treemap**
3. Data view: **Wazuh Alerts**
4. Group by:
    - First group: click field > **wazuh.rule.mitre.tactic**
        - Function: **Top values**
        - Number of values: **12**
    - Click **Add** to add a sub-group: **wazuh.rule.mitre.technique**
        - Function: **Top values**
        - Number of values: **5**
5. Metric: **Count of records**
6. Click **Save and return**

### Panel 1.6 - Top Blocked IPs (Data Table)

1. Click **Create visualization**
2. Visualization type: **Table**
3. Data view: **pfSense Filterlog**
4. Add filter in query bar: `event.action : "block"`
5. Columns:
    - Rows: **source.ip** > Top values > 15
    - Metric 1: **Count of records** (rename to "Blocks")
    - Add column: **source.geo.country_name** > Top values > 1 (rename to "Country")
6. Click **Save and return**


**Save the dashboard** as **SOC Overview**.
